75 Algorithms Description:
76 -----------------------
77 
78 The genetic algorithm consist of finding the best set of paramters for a given model using the following concepts
79 (shamelessly stolen from the work of our mother nature over the last several billions of years):
80 - <b>Population:</b> the genetic algorithm takes as input a population where each individual is a different instance
81 of a configurable model and evolve it into a population of individuals that best represent the solution to a given problem.
82 - <b>Variability:</b> variation is at the center of the genetic algorithms: if we have only a handfull of ways to configure
83 our model, the room for optimization towards a best solution would be limited. The configurations of the initial population
84 are randomly chosen to allow the exploration of the full phase space of configurations.
85 - <b>Selection and cross-over:</b> Some of the individuals in the original population will be better fitted than others to answer the
86 solution to our problem. A figure of merit is defined to provide a ranking of our population from the best to the least fitted.
87 The next generation is then constructed by crossing over individuals selected among the best fitted from the
88 current generation.
89 - <b>Mutations:</b> To maintain variability through the next generations, occasional mutations are introduced. These
90 consist of randomoly introducing new features (i.e. new configurations not inherited from the parents), with a controlled
91 rate.
92 
93 The genetic algorithm repeats the selection, cross-over and mutations steps until the best fitted individual reach a certain
94 fitness threshold.

98 Implementation details:
99 -----------------------
100 
101 ### The demo case:
102 
103 The demo case generates pseudo-random data following a gaussian distribution and uses the genetic algorithm
104 to guess the parameters of the gaussian from the data. The results are then compared with a likelihood fitting method
105 as implemented in the ROOT library.
106 
107 To make this implementation reusable in future projects, abstract interfaces are used for the different elements:
108 generic features are implemented in the interface base classes, while features that are specific to the demo case are
109 implemented in derived classes.
110 
111 ### The Model:
112 
113 The model is what we are trying to optimize. In general, it is described by a set of properties that can have different values.
114 - <b>The interface:</b> `IModel` is minimal since the details of the model are use-case specific. The only thing the interface
115 knows about is the fact that a model can be scored.
116 - <b>The demo case:</b> `ParametricModel` describes a parametric 1D formula of a gaussian distribution of the form:
117 \f[
118  f(x) = N e^{-\frac{(x-\mu)^2}{\sigma^2}}
119 \f]
120 The implementation using ROOT's powerful <a href="https://root.cern.ch/doc/v610/classTF1.html">TF1</a>
121 formula class allows our ParametricModel sub-class to describe any formula in any number of dimensions and depending on
122 any number of parameters, not to mention the possibility to draw a graph representing our model without the need for a
123 lot of extra code.
124 
125 
126 ### The figure of merit:
127 
128 Sometimes it's called a fitness function: I prefer the term figure of merit over fitness function, because the latter suggest
129 a function that increases with goodness (i.e. the higher the better) while there is no reason for this to always be the case.
130 In fact, in our demo case example, a better \f$\chi^2\f$ is a smaller one.
131 - <b>The interface:</b> `IFigureOfMerit` provides placeholders for the following functionalities:
132  - Assign a score to a given model.
133  - Given two scores, decides which is better.
134  - Given a score, decides if it passes the acceptable threshold criteria.
135 - <b>The demo case:</b> `Chi2FitFigureOfMerit` uses a \f$\chi^2\f$ over number of degrees of freedom estimator
136 to compare a model with a given data set.
137 A data set consists of \f$N\f$ \f$(\vec{x_i}, y_i)\f$ pairs with an associated error \f$\sigma_{y_i}\f$ on \f$y_i\f$.
138 The figure of merit is then given by:
139 \f[
140 \chi^2/nfd = \frac{1}{N}\sum_{i=0}^{N}\frac{(y_i - f(\vec{x_i}))^2}{\sigma_{y_i}^2}
141 \f]
142 
143 ### The population:
144 
145 The population defines a generation of models and controlls the way models are selected, crossed-over and mutated to build
146 the next generation.
147 - <b>The interface:</b> `IPopulation` already implements a large portion of the functionalities using the interfaces provided
148 for the figure of merit and the model. This include tasks such as ranking the population and finding the best fitted individuals,
149 selection based on this ranking, and handling the mutation rate.
150 - <b>The demo case:</b> `ParametricModelPopulation` implements the details of initialization, cross-over and mutation:
151  - Initialization: the initial population is constructed using random values for the parameters withing a pre-defined range.
152  - Cross-over: the cross-over of two individuals is achieved by passing on the values for each parameter randomly from one
153  or the oher parent. Parents are selected such as the probability to give offspring depends linearly on the rank of the parent.
154  Figure @ref Fig1 shows the probability vs rank for a population of 500. An exception is made to the best fitted individual
155  which is always preserved intact in the next generation.
156  - Mutation: the mutation is done by slightly modifying a randomly chosen parameter by adding a guassian noise component.

165 ### The algorithm:
166 
167 The `GeneticAlgorithm` class wraps everything above and controls the flow of the algorithm using the defined interfaces.
168 It performs the following steps:
169 - Creates an initial population.
170 - Rank the population.
171 - Repeat the following until a solution is found or a maximum numeber of generations is reached:
172  - Select parents among the fittest individuals.
173  - Cross them over to form a new population.
174  - Mutate some individuals.
175  - Rank the new population.
